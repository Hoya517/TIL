# TIL - MSA 마무리 정리 & Kubernetes 이해

## 1. 무조건 MSA가 정답일까?

### 1.1 MSA 도입 시 고려 사항

MSA는 다음과 같은 구성 요소가 함께 필요하다:

-   서비스 디스커버리 (Eureka)
-   API Gateway
-   Config Server
-   보안 (OAuth2, JWT)
-   분산 추적 (Zipkin)
-   이벤트 기반 통신 (Kafka)
-   모니터링 및 로깅

→ 단순한 구조가 아니라 **운영 체계 전체를 설계해야 하는 아키텍처**임.

------------------------------------------------------------------------

### 1.2 언제 모놀리틱이 더 나은가?

-   빠른 프로토타입이 필요할 때
-   팀 규모가 작을 때
-   도메인 경계가 명확하지 않을 때
-   운영 복잡도를 감당하기 어려울 때

MSA는 목적이 아니라 **수단**이라는 점이 중요.

------------------------------------------------------------------------

## 2. Kubernetes 이해

### 2.1 Kubernetes란?

-   컨테이너 오케스트레이션 플랫폼
-   배포, 확장, 복구 자동화
-   CNCF 관리 오픈소스

------------------------------------------------------------------------

### 2.2 핵심 개념

-   **Container**: 애플리케이션 실행 단위
-   **Pod**: Kubernetes 최소 배포 단위
-   **Node**: Pod가 실행되는 서버
-   **Cluster**: 여러 Node의 집합
-   **Namespace**: 리소스 논리적 분리 단위

------------------------------------------------------------------------

## 3. Spring Cloud vs Kubernetes

### 3.1 공통점

-   확장성 지원
-   고가용성
-   서비스 관리 자동화

------------------------------------------------------------------------

### 3.2 차이점

  구분   Spring Cloud              Kubernetes
  ------ ------------------------- --------------------------
  초점   애플리케이션 레벨         인프라 레벨
  구성   Eureka, Gateway, Config   Pod, Deployment, Service
  배포   JVM 기반 배포             컨테이너 기반 배포

------------------------------------------------------------------------

## 4. Spring Cloud Kubernetes

-   Kubernetes API 기반 서비스 디스커버리
-   ConfigMap, Secret 연동
-   클라우드 네이티브 환경 통합

------------------------------------------------------------------------

## 5. Kubernetes 장단점

### ✔ 장점

-   자동 스케일링
-   자동 복구
-   환경 독립성

### ✔ 단점

-   높은 학습 곡선
-   운영 복잡성
-   비용 증가 가능성

------------------------------------------------------------------------

## 6. 최종 정리

이번 강의를 통해 배운 것:

-   MSA는 단순 분리 아키텍처가 아니라 운영 중심 아키텍처
-   관측 가능성(Observability)이 핵심
-   인프라 자동화(Kubernetes)는 선택이 아닌 흐름
-   아키텍처는 팀 상황과 도메인에 맞게 선택해야 함

MSA는 기술 스택이 아니라 "조직과 운영을 포함한 설계 전략"이라는 점을
이해함.
